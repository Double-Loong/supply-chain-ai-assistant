# 智链AI - 供应链智能预测助手

**文档版本**：V1.0  
**撰写日期**：2025年5月  
**撰写人**：Double-Loong  
**产品定位**：基于AI LightGBM的SKU级需求预测与智能补货决策系统  
**目标用户**：供应链主计划员、库存管理员、采购专员  

---

## 一、项目背景

### 1.1 业务痛点

在Lotto实习进行SKU库存管理和在京东进行SKU日常运营时，我发现需求分析库存管理面临以下核心痛点：

**痛点1：预测依赖人工经验**

- 具体表现：许多工作用Excel手工调整，促销期预测偏差大
- 业务影响：缺货损失/库存积压

**痛点2：数据分散**

- 具体表现：订单、库存、用户数据在不同系统，难以关联分析
- 业务影响：决策滞后

**痛点3：补货决策慢**

- 具体表现：客户下单反馈需求后才执行缺货SKU补货
- 业务影响：周转效率低

**痛点4：缺乏量化评估**

- 具体表现：没有形成可预测的补货方式
- 业务影响：缺乏迭代优化

### 1.2 项目目标

构建AI驱动的需求预测原型系统，验证假设：融合时序特征与用户行为特征的机器学习模型，较传统移动平均方法能显著降低预测误差。

**成功指标**：

- 预测准确率（MAPE）：目标 < 20%，实际达成 **15.8%**
- 较传统方法提升：目标 > 20%，实际达成 **37.5%**
- 产品化演示：可交互仪表盘，**已完成**

## 二、用户画像

**用户1：供应链主计划员**

- 核心诉求：快速获得准确的SKU级销量预测
- 使用场景：每周一制定本周补货计划
- 痛点：Excel手工计算耗时2-3小时/周，准确率不稳定

**用户2：库存管理员**

- 核心诉求：提前识别库存风险，避免断货
- 使用场景：每日监控库存水位
- 痛点：预警滞后，经常被动补货

**用户3：采购经理**

- 核心诉求：基于数据制定采购量，减少资金占用
- 使用场景：月度/季度采购谈判
- 痛点：采购量凭经验，旺季缺货、淡季积压

## 五、核心功能设计

### 5.1 数据概览页

**页面布局**：

- 顶部导航：Logo + 菜单（概览/预测/补货/报告）+ 用户信息
- KPI卡片区：4个指标卡片（悬停显示环比变化）
- 趋势图区：近30天销量折线图（悬停显示具体数值）
- 预警列表：库存预警SKU列表（点击跳转预测页）
- 品类分布：品类销量柱状图（悬停显示占比）

**KPI指标定义**：

- 今日总销量：当日0:00-当前时间累计销量，实时更新
- 本周预测准确率：1 - |本周预测-实际|/实际，每日更新（>90%绿，80-90%黄，<80%红）
- 库存预警SKU数：库存<安全库存天数的SKU计数，每日更新（0绿，1-5黄，>5红）
- AI建议采纳率：采纳AI建议补货的次数/总建议次数，每周更新

### 5.2 预测分析页（核心）

**用户流程**：

进入页面 → 选择SKU → 选择预测周期 → 点击"开始预测" → 查看趋势图 → 查看预测详情表 → 跳转补货建议页

**控制面板**：

- SKU选择：下拉搜索框，支持按名称/编码搜索，默认最近查看
- 预测周期：单选按钮组（7天/14天/30天），默认7天
- 开始预测：主按钮，触发模型推理

**趋势图三条线**：

- 历史实际（蓝实线）：已发生的真实销量
- AI预测（绿虚线）：LightGBM模型输出
- 7日移动平均（黄点划线）：传统基线方法

**预测详情表字段**：日期 | AI预测（蓝高亮） | 传统法（灰） | 实际值 | AI偏差（绿=优于传统，红=差于传统）

### 5.3 补货建议页

**页面目的**：将预测转化为可执行的采购决策

**核心公式**：

```
预测总需求 = 日均预测销量 × 预测周期天数
安全库存 = 日均预测销量 × 安全库存天数
建议补货量 = max(0, 预测总需求 + 安全库存 - 当前库存)
```

**SKU选择器**：下拉搜索栏，品类选择，支持SKU编码和商品名称模糊搜索，输入时实时联想

**库存预警规则**：

- 当前库存 < 安全库存：红色警告，"当前库存仅够X天销售，低于安全线Y天。建议立即启动采购流程。"
- 安全库存 ≤ 当前库存 < 1.5×安全库存：黄色提醒，"库存处于警戒水位，建议关注未来补货计划。"
- 当前库存 ≥ 1.5×安全库存：绿色正常，"库存健康，可覆盖X天。建议持续监控。"

**预测周期选择**：7天/14天/30天，与预测页联动，修改时重新计算

**建议补货量**：页面视觉中心，48px大数字，>0 时红色（需补货），=0 时绿色（充足）

**生成采购单按钮**：模拟按钮，点击弹出"采购单已生成"提示，实际对接ERP时需开发接口

**补货优先级矩阵（四象限）**：

- 紧急补货（红）：高销量+低库存，立即采购，加急处理
- 关注周转（黄）：高销量+高库存，监控周转率，避免积压
- 常规补货（绿）：低销量+低库存，按标准流程补货
- 滞销风险（灰）：低销量+高库存，促销清仓，停止采购

### 5.4 模型报告页

**页面目的**：向技术面试官/数据团队证明AI价值

**品类选择器**：下拉搜索框，支持品类模糊搜索，输入时实时联想

**MAPE对比柱状图**：三个柱子：LightGBM、历史同期、移动平均，绿色最短，视觉冲击

**关键指标对比**：

| 模型 | MAE | RMSE | MAPE |
|------|-----|------|------|
| LightGBM-AI | 18.5 | 32.5 | 15.8% |
| 历史同期法 | 28.2 | 41.8 | 22.1% |
| 7日移动平均 | 32.1 | 45.2 | 25.3% |

**模型决策依据**：横向进度条，Top 5特征，证明模型可解释性

**模型提升效果**：大数字"37.5%"，副标题"LightGBM vs 传统方法 预测误差降低"，放在右侧，第一眼看到

**技术栈说明**：算法/数据/特征/训练方式，建立技术可信度

## 六、数据需求

### 输入数据

**order.csv**（40万条）：

- order_id（int）：订单唯一标识
- user_id（int）：用户标识
- product_id（int）：SKU标识
- order_time（datetime）：下单时间
- quantity（int）：购买数量
- amount（float）：订单金额
- promotion_type（string）：促销类型（None/满减/折扣等）
- order_status（string）：订单状态（Delivered/Cancelled等）
- shipping_city（string）：收货城市
- product_name（string）：商品名称
- category（string）：品类
- brand（string）：品牌
- price（float）：单价
- is_hot（int）：是否热销（0/1）

**user.csv**（10万条）：

- user_id（int）：用户标识
- gender（string）：性别
- age（int）：年龄
- register_time（datetime）：注册时间
- total_purchase_times（int）：累计购买次数
- total_purchase_amount（float）：累计购买金额
- last_purchase_time（datetime）：最近购买时间

### 输出数据

**daily_sales.csv**：

- order_date（date）：日期
- product_id（int）：SKU
- daily_qty（int）：日销量
- daily_amount（float）：日销售额
- daily_orders（int）：日订单数
- daily_users（int）：日购买用户数

**predictions.csv**：

- order_date（date）：预测日期
- product_id（int）：SKU
- pred_lgb（float）：LightGBM预测值
- pred_ma（float）：移动平均预测值
- actual（float）：实际值（到达日期后回填）
